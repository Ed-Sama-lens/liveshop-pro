1 รอบ 1 — ธงปิด (ค่า default จริง)

npm run dev
รอจนเห็น Local: http://localhost:3000 (terminal จะค้างแบบนี้ = ปกติ ห้ามปิด)

เปิด Chrome → http://localhost:3000/sale → login admin → เช็ค:

  • 1เห็น panel "รหัสสินค้า" แบบเดิม (legacy) ครบ
  • 2ไม่เห็น กล่อง "Sale Board (V Rich style) — preview"
  • 3หน้าตาเหมือนเดิมทุกอย่าง ไม่มีอะไรเพี้ยน
เสร็จรอบ 1 กลับไป terminal กด Ctrl+C หยุด server

2 รอบ 2 — ธงเปิด (หัวใจของการทดสอบ ⭐)

พิสูจน์ว่า board ใหม่โผล่ และเป็น อ่านอย่างเดียว จริง (ไม่ยิงคำสั่งแก้ข้อมูล)

Git Bash 
NEXT_PUBLIC_SALE_LAYOUT_V2=true npm run dev
PowerShell 
$env:NEXT_PUBLIC_SALE_LAYOUT_V2='true'; npm run dev

refresh http://localhost:3000/sale → เช็ค 7 จุด:

  • 1panel "รหัสสินค้า" เดิม — ยังอยู่ ไม่หาย
  • 2กล่อง "Sale Board (V Rich style) — preview" โผล่เพิ่มขึ้นมา
  • 3คลิก pill (ชิปรหัสสินค้า) → drawer/รายละเอียดเปิด
  • 4ใน drawer — อ่านอย่างเดียว ไม่มีปุ่มบันทึก/ยกเลิก/ช่องกรอก
  • 5จุดสำคัญสุด DevTools Network — คลิก pill แล้ว ห้ามมี POST/PUT/PATCH/DELETE ไป /api/CRITICAL board นี้รับข้อมูลมาแล้วจาก server (ไม่ดึงเอง) — คลิก pill ปกติจะ ไม่เห็น request ใหม่เลย ก็ถูก. วิธีดูละเอียด ↓
  • 6กด F12 → Console — ไม่มี error สีแดง
  • 7กด Ctrl+Shift+M (มือถือ) เลือก iPhone — board ไม่ล้นจอ อ่านได้
🔑 วิธีดูจุดที่ 5 ให้ละเอียด (สำคัญสุด — กันแก้ข้อมูลจริงโดยไม่ตั้งใจ)
1กด F12 เปิด DevTools
2คลิกแท็บ Network
3กดปุ่ม filter Fetch/XHR
4เคลียร์รายการให้ว่างก่อน (ปุ่ม 🚫 หรือ Ctrl+L)
5คลิก pill ในบอร์ดหลายอัน เปิด drawer ดู
6ดูคอลัมน์ Method: GET = ปกติ · POST/PUT/PATCH/DELETE ไป /api/ = FAIL

  • 1กด Ctrl+C หยุด server
  • 2รัน npm run dev ธรรมดา (ไม่มีธง)
  • 3refresh /sale → board "V Rich" หายไป เหลือแต่ legacy เหมือนรอบ 1
